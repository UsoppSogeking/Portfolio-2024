Pokedéx

Minha expêriencia com esse projeto:

Criei essa sessão para compartilhar minhas dificuldade, aprendizados e experiência durante o desenvolvimento do projeto Pokedex. Caso queira ir direto ao projeto clique aqui.


Decidi criar este projeto com essa temática de pokemon por dois motivos, o primeiro é que eu amo animes/mangás e o segundo é que eu queria aprender React, encontrei a api nesse site, como eu ja havia criado um projeto que consumia uma rest api externa e já tinha uma boa noção de como o consumo de dados funciona optei por criar o projeto em React, que é uma tecnologia muito requisitada no mercado de trabalho.


Antes de começar o projeto pesquisei mais algumas tecnologias muito usadas no dia a dia de programadores, algumas como react-router-dom eu já conhecia devido a um curso que fiz, e encontrei uma lib chamada axios, que é usada para consumir apis, inicialmente achei o axios uma lib bem amigável e no fim das contas ele realmente é muito simples de entender, por outro lado a api que escolhi trabalhar não era tão amigável assim, diferente da api rick and morty ela é bem verbosa, não é tão facil acessar dados especificos como foi com a api do projeto RaM, sua documentação também não é tão intuitiva e possui endpoints demais, que eu particurlamente achei desnecessário, pois, para acessarmos todas as infos de um pokemon precisamos fazer requisições para MUITOS endpoints.


Aprendizados: Ao decorrer do projeto pude entender melhor o uso dos hooks useState e useEffect que antes eram um pesadelo para eu entender, também fiz muito uso do css.module e layout responsivo e a medida que eu ia avançando no projeto tudo foi ficando mais natural e eu entendi que React, assim como programação no geral, não é tão dificil, mas sim questão de estudo e prática aplicados, tabém aprendi a compartilhar dados entre componentes atrávez de props, que era uma outra grande dificuldade que eu tinha quando comecei a estudar React, muito do que aprendi no projeto com api Rick and Morty foi muito util para o desenvolvimento deste projeto, também aprendi a alterar a cor de um elemento html de acordo com um dado especifico da api, no caso o dado desse projeto é o elemento do pokemon


Dificuldades: Mesmo conseguindo fazer uso dos conhecimentos obtidos com o projeto Rick and Morty ainda tive dificuldades com este projeto, como eu já havia dito essa api é muito verbosa e acessar os dados não eram tão simples como na api RaM e demorei um tempo até conseguir acesso a eles para fazer a listagem na Home, também tive problemas em apresentar os dados individuais de cada pokemon ao clicar em seu card, o que me levou a refazer o projeto pelo menos 4 vezes, mas o bom de tudo isso é que refazendo ele algumas vezes pude fixar diversos conhecimentos o que tornou as coisas mais faceis posteriormente, outra dificuldade foi conseguir tratar os dados de peso e altura, ainda estou estudando sobre, mas os dados de altura foram fornecidos em cm e os dados de peso foram fornecidos incompletos, o que me fez ter que transforma-lo para o formato de gramas. #Nota: enquanto eu terminava de escrever os pontos de maiores dificuldade acabei pensando em como tratar os dados de peso e altura e após revisar o projeto consegui chegar na solução, que no final das contas era muito mais simples do que eu havia pensado e tentado implementar, então provavelmente ao acessar o projeto os dados já estarão atualizados para o formato correto.


Solução do tratamento de dados: Ambos os dados de peso e altura foram fornecidos de forma "incorreta", o dado de peso do bulbasaur por exemplo foi fornecido como 69, enquanto que sua altura foi fornecida como 7, então apenas multipliquei o peso fornecido por 100 para obter o valor em gramas(6900g) e depois dividi por 1000 para obter o valor em kg(6,9kg), quanto a altura, multipliquei por 10 para obter o valor em cm(70cm) e depois dividi por 100 para obter o valor em metro(0,7m).


Próximos passos: Agora que já tenho um bom conhecimento de React pretendo fazer uma nova aplicação usando a api Rick and Morty, provavelmente vou reutilizar o layout da aplicação em vanilla e também tecnologias como redux, typescript e talvez sass ou bootstrap. Tenho que estudar essas tecnologias antes é claro apesar de já ter visto pelo menos duas delas no curso que fiz e quando acabar pretendo refazer meu portfólio com React.
